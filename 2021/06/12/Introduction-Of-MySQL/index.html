Databases and MySQL

Structured Query Language

the language that we use to issue commands to the database

  • Create/Insert
  • Read/Select
  • Update
  • Delete

Terminology

Database

Contains one or more tables

Table (or relation)

contains tuples (row) and attributes (column/field)

Tuple

a set of fields which generally represent an “object” like a person or a music track

Attribute

one of possibly many elements of data corresponding to the object represented by the row

Common DBSM

  • Oracle

    Large, commercial, enterprise-scale, very tweakable

    • MySQL

      Simpler but very fast and scalable - commercial open source

    • SqlServer

      Very nice - from Microsoft (also Access)

Basic SQL Operations

Command Line

  • Macintosh

    /Applications/MAMP/Library/bin/mysql --host=localhost -uroot –p

    Enter “root” when prompted for the password

  • Windows

    c:\xampp\mysql\bin\mysql.exe -u root –p

    Press enter when prompted for password

Show Database

the function to print out the content of the database.

Create Database

to create new database

Create Table

to create a new table

Drop Table

Insert Row

The INSERT statement inserts a row into a table

Delete Row

Deletes a row in a table based on selection criteria

Update Row

Allows the updating of a field with a WHERE clause

Select Row

Retrieves a group of records - you can either retrieve all the records or a subset of the records with a WHERE clause

Order By

You can add an ORDER BY clause to SELECT statements to get the results sorted in ascending or descending order

like

We can do wildcard matching in a WHERE clause using the LIKE operator

Limit

LIMIT 2 is same as LIMIT 0,2

LIMIT n,m ‘n’ means the select begin from n index in table; ‘m’ means how many number of rows will be selected after n index.

Count()

return the number of selected items

Data Types

String Fields

  • CHAR allocates the entire space (faster for small strings where length is known)
  • VARCHAR allocates a variable amount of space depending on the data length (less space)

Text Fields

  • TINYTEXT up to 255 characters
  • TEXT up to 65K
  • MEDIUMTEXT up to 16M
  • LONGTEXT up to 4G

Binary Types

  • BYTE(n) up to 255 bytes
  • VARBINARY(n) up to 65K bytes

Binary Large Object

  • TINYBLOB(n) - up to 255
  • BLOB(n) - up to 65K
  • MEDIUMBLOB(n) - up to 16M
  • LONGBLOB(n) - up to 4G

Integer Numbers

  • TINYINT (-128, 128)
  • SMALLINT (-32768, +32768)
  • INT or INTEGER (2 Billion)
  • BIGINT - (10**18 ish)

Floating Point Numbers

  • FLOAT (32-bit) 10**38 with seven digits of accuracy
  • DOUBLE (64-bit) 10**308 with 14 digits of accuracy

Dates

  • TIMESTAMP - ‘YYYY-MM-DD HH:MM:SS’ (1970, 2037)
  • DATETIME - ‘YYYY-MM-DD HH:MM:SS’
  • DATE - ‘YYYY-MM-DD’
  • TIME - ‘HH:MM:SS’
