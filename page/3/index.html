Basic concepts

User Interface:

Front-end application view to which user interacts in order to use the software. User can manipulate and control the software as well as hardware by means of UI.

Usability:

Usability is a quality attribute (see chapter 2 in S.Arch) that assesses how easy user interfaces are to use.

  1. Learnability: How easy is it for users to accomplish basic tasks the first time they encounter the design?

  2. Efficiency: Once users have learned the design, how quickly can they perform tasks?

  3. Memorability: When users return to the design after a period of not using it, how easily can they re-establish proficiency?

  4. Errors: How many errors do users make, how severe are these errors, and how easily can they recover from the errors?

  5. Satisfaction: How pleasant is it to use the design?

User Experience:

User experience” encompasses all aspects of the end-user’s interaction with the company, its services, and its products (Nielsen).

Main Aspect of UX

  1. Subjectivity: UX exists in the mind of the user, not in the object (or artefact).
  2. Contributing agents: UX is the consequence of interaction between the following three contributors: object, user, and context.
  3. Singular and accumulated experiences: Singular experiences is a result of an interaction with a touch-point, whereas the accumulated experience is the result of all singular experiences.

Interaction Design (IxD)

the design of interactive products and services in which a designer’s focus goes beyond the item in development to include the way users will interact with it.

5 Dimensions

  1. Words
  2. Visual representations
  3. Physical objects or space
  4. Time
  5. Behaviour

Annotations in Java

2021-03-22

Annotations are used to provide supplement information about a program.

What is Annotations

Annotations are also called metadata,a data used to describe data, such as, @Override and @Deprecated.

It was imported in JDK1.5 and used to explain the code.

  • Annotations start with ‘@’.
  • Annotations do not change action of a compiled program.
  • Annotations help to associate metadata (information) to the program elements.
  • Annotations are not pure comments as they can change the way a program is treated by compiler.

Usages

  1. Creating javadoc Documents via metadata from annotations in code.
  2. Checking compile
  3. Dynamic processing at compile time
  4. Dynamic processing at runtime

Categories

  1. Java default annotations

    Including @Override, @Deprecated (Indicate that a class or method is out of date) and @SuppressWarnings (Mark the warnings to be ignored)

  2. Meta annotations

    meta annotations are annotations used to define annotations.

    Including @Retention (indicating the stage at which the annotation is retained), @Target (indicating the scope of the annotation use), @Inherited (indicating that the annotation can be inherited), @Documented (indicating whether to generate a javadoc document)

  3. Customize

    Defining by user

Default annotations

@Deprecated

Indicating the elements which are out of date. The warning will be throwed by compiler to tell developer that he is calling a out-of-date element.

@Override

Mark that this method covers the method of the parent class.

@SuppressWarnings

Used to selectively turn off the compiler’s warnings for classes, methods, member variables, and variable initialization.

@SafeVarargs

Parameter safety type annotation. Its purpose is to remind developers not to use parameters to do some unsafe operations. Its existence will prevent the compiler from generating unchecked warnings. It was added in the Java 1.7.

@FunctionalInterface

https://docs.oracle.com/javase/8/docs/api/java/lang/FunctionalInterface.html

Meta annotations

@Retention

The RetentionPolicy.SOURCE annotation is only reserved in the source code phase, and it will be discarded and ignored when the compiler compiles.

The RetentionPolicy.CLASS annotation is only retained until compilation is in progress, and it will not be loaded into the JVM. Such as Java built-in annotations, @Override, @Deprecated, @SuppressWarnning, etc.

The RetentionPolicy.RUNTIME annotation can be retained until the program is running, it will be loaded into the JVM, so they can be obtained when the program is running. Such as @Controller, @Autowired, @RequestMapping, etc. in SpringMvc.

@Documented

Its function is to be able to include the elements in the annotations into the Javadoc.

@Target

  • ElementType.ANNOTATION_TYPE can annotate an annotation
  • ElementType.CONSTRUCTOR can annotate the construction method
  • ElementType.FIELD can annotate attributes
  • ElementType.LOCAL_VARIABLE can annotate local variables
  • ElementType.METHOD can annotate methods
  • ElementType.PACKAGE can annotate a package
  • ElementType.PARAMETER can annotate the parameters in a method
  • ElementType.TYPE can annotate a type, such as class, interface, enumeration

@Inherite

This is a annotation for annotations. It means that subclasses of annotated classes are considered having the same annotation as their superclass.

@Repeatable

Repeatable usage scenarios: When you need to use the same annotation multiple times, you often need to use @Repeatable.

Identifier

A Java identifier is a name given to a package, class, interface, method, or variable. It allows a programmer to refer to the item from other places in the program.

Rules for defining Java Identifiers

  • The only allowed characters for identifiers are all alphanumeric characters([A-Z],[a-z],[0-9]), ‘$‘(dollar sign) and ‘_‘ (underscore).
  • Identifiers should not start with digits([0-9]).
  • Java identifiers are case-sensitive.
  • There is no limit on the length of the identifier but it is advisable to use an optimum length of 4 – 15 letters only.
  • Reserved Words can’t be used as an identifier.

Original code

Convert an integer to a binary format

E.g:

One’s complement

One’s complement

The inverse code of a positive number: the same as the original code

The inverse code of a negative number: the sign bit of the original code remains unchanged, and the other bits are inverted

For example, the complement of -2 is: 11111111 11111111 11111111 11111101

Two’s Complement

The complement of a positive number: the same as the original code

The complement of a negative number: one’s complement +1

For example, the complement of -2 is: 01111111 11111111 11111111 11111110

Precautions

The highest bit of the binary system is the sign bit (“0” represents a positive number, and “1” represents a minus number);

Variable

The data which can be changed is saved in memory

Category

Member variable

Variables defined outside the method and inside the class.

  1. static variable

    • most of the static variables are visible. (public)
    • Static variables are created when they are accessed for the first time and destroyed at the end of the program.

  2. non-static variable

    • most of the non-static variables are invisible. (private)
    • Instance variables are created when the object is created, and destroyed when the object is destroyed

  3. Difference

    Static variable Non static variable
    Static variables can be accessed using class name Non static variables can be accessed using instance of a class
    Static variables can be accessed by static and non static methods Non static variables cannot be accessed inside a static method.
    Static variables reduce the amount of memory used by a program. Non static variables do not reduce the amount of memory used by a program
    Static variables are shared among all instances of a class. Non static variables are specific to that instance of a class.
    Static variable is like a global variable and is available to all methods. Non static variable is like a local variable and they can be accessed through only instance of a class.
    Static variables are stored in the static area in the method area. Non static variables are stored in heap memory.

Local variable

Variables in the methods of the class.

Local variables are declared in methods, construction methods, or statement blocks;

Local variables are created when methods, constructors, or statement blocks are executed. When they are executed, the variables will be destroyed;

Access modifiers cannot be used for local variables;

Local variables are only visible in the method, construction method, or statement block that declares it;

Local variables are allocated on the stack.

Local variables have no default value, so after the local variables are declared, they must be initialized before they can be used.

Java Introduction

2021-03-18

What is Java

Java (programming language)

Java is a class-based, object-oriented programming language.

Java (software platform)

Java is a system for developing application software and deploying it in a cross-platform computing environment.

Background

Java was started as a project called “Oak” by James Gosling in June 1991. Gosling’s goals were to implement a virtual machine and a language that had a familiar C-like notation but with greater uniformity and simplicity than C/C++. The first public implementation was Java 1.0 in 1995.

Versions

Java SE (J2SE, Java 2 Platform Standard Edition)

Java SE is a computing platform for development and deployment of portable code for desktop and server environments.

Java EE (J2EE, Java 2 Platform Enterprise Edition)

Jakarta EE, formerly Java Platform, Enterprise Edition (Java EE) is a set of specifications, extending Java SE with specifications for enterprise features such as distributed computing and web services.

Java ME (J2ME, Java 2 Platform Micro Edition)

Java ME is a computing platform for development and deployment of portable code for embedded and mobile devices (mobile phones, TV set-top boxes, etc.).

Features

Simple

Java is easy to learn and its syntax is quite simple, clean and easy to understand

There’re no pointers and operator overloading in Java.

Object Oriented

In java, everything is an object.

Object

A Java object is a combination of data and procedures working on the available data.

Door has states (open/close) and behaviours (pull()/push())

Class

A Class is like an object constructor, or a “blueprint” for creating objects.

Jenny and Danny are Students.

*Inheritance

Inheritance can be defined as the process where one class acquires the properties (methods and fields) of another

Father and Son

*Polymorphism

Polymorphism means “many forms”, and it occurs when we have many classes that are related to each other by inheritance.

*Encapsulation

In encapsulation, the variables of a class will be hidden from other classes, and can be accessed only through the methods of their current class.

private

*Java object-oriented programming has 3 major characteristics: encapsulation, inheritance, and polymorphism.

Robust

Robust simply means strong. Java is robust because: It uses strong memory management.

Platform Independent

The meaning of platform-independent is that the java compiled code(byte code) can run on all operating systems.

Secure

Java program always runs in JRE with almost null interaction with system OS, hence it is more secure.

Multi-Threading

Java multithreading feature makes it possible to write program that can do many tasks at the same time.

Architectural Neutral

Java program is easy to interpret on any machine.

Portable

Java Byte code can be carried to any platform.

High Performance

Java is an interpreted language, so it cannot be as fast as a compiled language like C or C++. However, Java enables high performance with the use of just-in-time compiler.

Distributed

Programs can be designed to run on computer networks. Java has a special class library for communicating using TCP/IP protocols.

Java Technological System

JVM, JRE & JDK

Overview

We must understand the differences between JDK, JRE, and JVM before proceeding further to Java.

Brief Introduction

JVM

JVM (Java Virtual Machine) is an abstract machine.

It doesn’t physically exist.

It provides a runtime environment in which Java bytecode can be executed.

It can also run those programs which are written in other languages and compiled to Java bytecode.

JRE

JRE is Runtime Environment.

A set of software tools which are used for developing Java applications.

It is used to provide the runtime environment.

It is the implementation of JVM.

It includes a set of libraries and other files that JVM uses at runtime.

JDK

JDK(Java Development Kit)

is a software development environment which is used to develop Java applications and applets.

It physically exists.

It contains JRE and development tools.
